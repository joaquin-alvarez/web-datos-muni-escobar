# Considerationes para desarrollo pre-producción

## Requisitos Legales para el Portal que debe cumplir
1. El Portal debe contar con un glosario.
2. Metadatos sobre cada dataset, es decir grupos de datos que describen el contenido informativo de un recurso o datasets (La fecha de creación, qué organismo es responsable de la base, periodicidad, la versión de la base datos, etc.)
3. Debe incluir una API que refleje los mismos datos que se publican en el portal de datos abiertos.
4. Debe incluir un espacio/sección o link a donde el ciudadano pueda solicitar información online adicional.
5. Debe incluir una sección con información del intendente/a y del gabinete (foto, biografía y cv).
6. Debe incluir una lista/s de los funcionarios con referencia al área al que pertenecen, la categoría y el cargo. Parcial, hasta secretarios. (Obligatorio).
7. Debe incluir una lista con los organismos y entidades del estado con su estructura orgánica y funciones - Parcialmente, hasta secretarios.
8. Debe incluir una lista con los datos de contacto de las áreas del gobierno - Parcialmente, hasta secretarios + contacto general del área.

*Opcional*:
9. Información en tiempo real y actualizaciones automáticas.

## Sobre el desarrollo:
1. Implementar todas las funcionalidades anteriores.
2. Crear data mock para los datos que no se encuentran disponibles.
3. Utilizar placeholders para imagenes y texto lorem ipsum.
4. Planificar el desarrollo de manera que se pueda probar en entorno de desarrollo.
5. Planificar posibles arquitecturas para el desarrollo de una API que refleje los mismos datos que se publican en el portal de datos abiertos. 
6. Planificar posibles arquitecturas para el desarrollo de un backoffice que permita la gestión de los datos.
